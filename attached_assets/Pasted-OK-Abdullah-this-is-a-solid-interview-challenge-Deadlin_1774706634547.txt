### STEP 2: The AI Pipeline Architecture (This is what they evaluate hardest)

This is a **multi-step pipeline** — not just "send image to GPT." Each step extracts different signals:

```
User uploads image + optional prompt
              ↓
    ┌─────────────────────┐
    │  STEP A: Upload to   │
    │  Azure Blob Storage   │
    └──────────┬────────────┘
              ↓
    ┌─────────────────────┐
    │  STEP B: Azure Vision │  ← Image Analysis 4.0
    │  Extract visual signals │
    │  - Dense captions       │
    │  - Tags & objects       │
    │  - OCR (text in image)  │
    │  - Color analysis       │
    │  - Brand detection      │
    └──────────┬────────────┘
              ↓
    ┌─────────────────────┐
    │  STEP C: Rule Parser  │  ← Parse Rules PDF → structured JSON
    │  (one-time on startup) │     (cached in memory/Redis)
    └──────────┬────────────┘
              ↓
    ┌─────────────────────┐
    │  STEP D: GPT-4.1     │  ← Compliance Reasoning Engine
    │  Input: visual signals │
    │       + parsed rules   │
    │       + user prompt     │
    │  Output: structured     │
    │    compliance JSON      │
    │  temp=0, seed=42        │
    └──────────┬────────────┘
              ↓
    ┌─────────────────────┐
    │  STEP E: Response     │
    │  Structured result:    │
    │  - Pass/Fail/Warning   │
    │  - Issues found        │
    │  - Rules violated      │
    │  - Fix suggestions     │
    │  - Confidence scores   │
    └───────────────────────┘
```

This multi-step pipeline is a **bonus challenge** they mentioned — and it shows serious AI engineering.

### STEP 5: Extra High-Tech Features (What Will Impress Them)

Beyond the base requirements, add these:

**5a. Rule Parser (PDF → JSON)** — On startup, parse the Test_rules.pdf into structured JSON with categories like logo_rules, color_rules, typography_rules, gradient_rules, background_rules. Store in memory. This is a bonus challenge they listed.

**5b. Confidence Scoring** — Each compliance check gets a confidence score (0-100%). Show it as a visual meter in the UI.

**5c. Visual Annotations** — Use Azure Vision's bounding box data to highlight problem areas on the uploaded image (overlay boxes where violations detected).

**5d. Side-by-side Comparison** — Show the uploaded image alongside the relevant rule section that was violated, so the user can visually compare.

**5e. Consistency Verification** — Store analysis results with a hash of the image. When the same image is re-uploaded, show both results side-by-side to prove consistency. This directly addresses their "Consistency Requirement."

**5f. Real-time Streaming** — Stream the GPT-4.1 analysis response to the frontend as it generates, so the user sees results appear progressively. Use FastAPI's StreamingResponse + SSE on the frontend.

**5g. Dark/Light Mode** — Simple but polished UI touch with shadcn/ui theming.
